# Example taxonomies from MAST-Data

These taxonomies were generated by ATLAS on traces from
[MAST-Data](https://huggingface.co/datasets/mcemri/MAST-Data), a public
dataset of failure traces from a variety of multi-agent systems and
benchmarks. Each file is a real, end-to-end taxonomy produced by running
the ATLAS pipeline over a single (multi-agent system, benchmark)
combination.

| File | Multi-agent system | Benchmark | Underlying LLM |
| --- | --- | --- | --- |
| `AG2_GSMPlus.json` | AG2-MathChat | [GSM-Plus](https://qtli.github.io/GSM-Plus/) | GPT-4 |
| `AG2_MMLU.json` | AG2-MathChat | [MMLU](https://github.com/hendrycks/test) | GPT-4o-mini |
| `AG2_OlympiadBench.json` | AG2-MathChat | [OlympiadBench](https://github.com/OpenBMB/OlympiadBench) | GPT-4o |
| `AppWorld_TestC.json` | [AppWorld](https://appworld.dev/) | AppWorld Test-C | GPT-4o |
| `ChatDev_ProgramDev.json` | [ChatDev](https://github.com/OpenBMB/ChatDev) | Program development | GPT-4o |
| `HyperAgent_SWEBench.json` | [HyperAgent](https://github.com/FSoft-AI4Code/HyperAgent) | [SWE-Bench Lite](https://www.swebench.com/) | Claude-3.7-Sonnet |
| `MagenticOne_GAIA.json` | [Magentic-One](https://github.com/microsoft/autogen/tree/main/python/packages/autogen-magentic-one) | [GAIA](https://huggingface.co/gaia-benchmark) | GPT-4o |
| `MetaGPT_ProgramDev.json` | [MetaGPT](https://github.com/geekan/MetaGPT) | Program development | GPT-4o |
| `OpenManus_GAIA.json` | [OpenManus](https://github.com/mannaandpoem/OpenManus) | GAIA | GPT-4o |

Each file follows the ATLAS taxonomy schema with three categories:

- **`category_a`** — system failures (agent-independent: truncation, timeouts, looping, ...)
- **`category_b`** — role-specific quality failures (`Solver_*`, `Checker_*`, `Refiner_*`, ...)
- **`category_c`** — domain reasoning failures (sign errors, missed cases, wrong proof structure, ...)

Each code is a dict with `code`, `name`, `definition`, `when_to_use`,
`when_not_to_use`, `detection_heuristics`, `severity`, and (for A codes)
`evidence: theoretical | observed`. These files are the compact "short"
form — for the full pipeline output (architecture, role definitions,
trace signals, etc.) re-run ATLAS against the source traces in MAST-Data.

## How they were generated

```bash
# 1. Download a slice of MAST-Data (see https://huggingface.co/datasets/mcemri/MAST-Data)
# 2. Run ATLAS against it:
python -m atlas generate \
    --traces path/to/AG2-MathChat_GSMPlus_traces.jsonl \
    --output ./AG2_GSMPlus_out \
    --max-codes 25
```

The `taxonomy.json` produced is structurally identical to the files in this
directory (modulo the `full_layer` / `annotation_layer` split — these files
correspond to the `annotation_layer.category_*` view).

## Using them

These taxonomies are drop-in usable with `atlas.classify_trace`:

```python
from atlas import classify_trace, load_traces

trace = load_traces("my_new_trace.json", verbose=False)[0]
diagnosis = classify_trace(
    "examples/mast_data_taxonomies/AG2_GSMPlus.json",
    trace,
)
print(diagnosis.code, diagnosis.label)
```

You can also use them as a head start: pick the taxonomy closest to your
system and refine on your own traces.
